基因组浏览器 JBrowse 安装（基于 Nginx 的 Web 服务器）

2020-10-03 | BioinformaticsLinux

需求：将基因组浏览器 JBrowse 安装在 Linux 服务器，在内网中用客户端电脑的网页浏览器可以打开 JBrowse 进行操作。安装过程如下，以 CentOS 7 系统为例。

1 先安装 Jbrowse 依赖的软件

参考资料：https://jbrowse.org/docs/installation.html

需要先安装依赖的一些工具，即“system pre-requisites”。我用的centOS 7系统，操作如下：

安装 node 和 npm
源代码下载 https://nodejs.org/dist/v12.18.4/node-v12.18.4-linux-x64.tar.xz
xz文件解压：tar -xvJf node-v8.11.1-linux-x64.tar.xz
解压之后文件目录里面包含bin目录（node，npm，npx 三个程序），将bin目录加入环境变量。
export PATH=/xxx/xxx/bin:$PATH
source ~/.bashrc 之后即可测试 node -h 与 npm -h，均可以运行。

2 下面安装Jbrowe

运行 setup.sh 自动安装成功，setup.log 文件如下所示。

到这一步是无法从其他客户端访问服务器上的JBrowse的，需要在服务器上面安装Web服务器。比如 appache/xampp/nginx等。

xampp案例见：https://www.jianshu.com/p/1dddec8dc29c
nginx案例见：https://yq.aliyun.com/articles/650480

3 安装 Nginx Web服务器

我使用nginx做测试的。下载ngnix源文件，root权限下，解压在/usr/local/ngnix下，./configure && make && make install 成功安装nginx（默认web端口是80），然后就可以在客户端浏览器输入服务器地址，看到初始页面：

上方的网页截图，实际上是由nginx安装目录下的nginx.conf配置的，红色标记处，location指向了/usr/local/nginx/html/index.html

nginx安装参考：https://blog.csdn.net/t8116189520/article/details/81909574 （实测：修改端口号之后无法访问）

`## 停止nginx
/usr/local/nginx//sbin/nginx -s stop

重启（reload）如果报错，就要先输入下面的命令`

`/usr/local/nginx/sbin/nginx -c /usr/local/nginx/conf/nginx.conf

重启 nginx`

../sbin/nginx -s reload

那么如何将 Jbrowse 相关文档经过 Nginx 平台呈现在客户端的web界面？需要修改 nginx的配置文件：“/usr/local/nginx/conf/nginx.conf”，root权限下vim这个配置文件，在 http 内容里面家加入下面的文字。端口号可以自己设置，比如8080、8082等。location指向/var/www/html/jbrowse/index.html

查询资料的过程中发现其他人安装nginx之后有这两个目录“/etc/nginx/sites-available/ 和 /etc/nginx/sites-enabled”，我安装之后是没有的，后面发现并没有影响。

server {
listen 8000;
server_name 192.168.1.108;
location / {
root /var/www/html/jbrowse;
index index.html;
}

按照上图修改并保存，重启 nginx 理论上可以在客户端打开192.168.1.108:8000即可看到jbrowse，可是我遇到了打不开的问题。原来不仅需要在 nginx 里面设置端口，还需要linux系开放相应的端口：

iptables -A INPUT -ptcp –dport 8000 -j ACCEPT

然后就顺利打开了
