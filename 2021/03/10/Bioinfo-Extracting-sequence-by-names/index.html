2021-03-10 | Bioinformatics

依据序列名称从fasta文件提取序列是一个很常见的操作。seqkit 这个工具可以很方便地实现。
比如 input.fa 包含若干条序列，名称分别为 1、2、3……100 。而 name list 文件包含 40——49 这十个ID，那么通过下面的代码就可以提取编号 40——49 对应的序列。

上面只是一种简单的情况，而大部分原始 fasta 文件中，序列名除了 ID，还可能会包含其他信息（比如基因注释）。当输入的 name list 文件只有 ID，默认的“名称完全匹配”就提取不了，就需要加上 -r 参数来申明使用正则匹配（regular expression）进行提取。

另一种情况就是，对于拟南芥，每个基因有多个 isoforms，如果我们只需要依据基因 ID 输出一个 isoform，那么可以通过 --delete-matched 参数来控制，即当某个 ID 出现了一次匹配，则立即停止继续检索该 ID，进入下一个 ID 检索。
